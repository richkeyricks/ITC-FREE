This release focuses on security hardening, rate limiting, and optimization of the handshake and MT5 verification loops to ensure session isolation and stable runtime performance.

### ⚡ The Security Hardening & Handshake Speedup Updates
*   🛡️ Supabase Client Rate Limiter: Implemented a rate limiting mechanism on client connection recreation (recreate_client) to prevent connection pool storms during transient network drops or SSL protocol violations.
*   🔒 Plaintext Password Storage Removal: Removed the plaintext saved_password field from the database profile synchronization (ensure_profile_exists) to comply with strict security standards.
*   🔐 Session State Isolation: Enhanced session authorization security by shifting local permission evaluations to secure runtime-only memory states instead of relying on persistent local disk variables.
*   🛡️ Session Verification Enforcement: Corrected startup routing flow to require active authentication state validation directly with the cloud service provider, replacing unverified local cached session IDs.
*   🧹 Session Cleanup on Logout: Enhanced the logout routine to completely wipe all cached profiles, verified session states, runtime environment variables, and local identity tokens to ensure multi-user isolation on public deployments.
*   🚀 Handshake Sequence Speedup: Optimized boot performance by reducing hardcoded time.sleep intervals inside the _execute_handshake loop from a minimum of 6.2 seconds down to 1.7 seconds.
*   ⚡ MT5 Connection Pulse Speedup: Reduced the connection verification retries inside the handshake pulse step from 5 attempts (1.0s interval) to 2 attempts (0.5s interval), reducing freeze duration when the MT5 terminal is offline.
