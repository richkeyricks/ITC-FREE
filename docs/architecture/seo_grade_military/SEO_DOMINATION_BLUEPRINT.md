## üõ∞Ô∏è PHASE 11: MULTI-ENGINE SYNCHRONIZATION (BING & BEYOND)
*Google is the king, but Bing/AI Search (Perplexity) is the rising empire.*

### üîç 11.1 Bing & IndexNow
Every modern Haineo site must support **IndexNow**.
- **Implementation**: Generate an IndexNow key and ping `https://www.bing.com/indexnow` every time content changes.
- **Bing Webmaster Tools**: Use the "Import from GSC" feature but manually verify sitemaps once a month.

### ü§ñ 11.2 The Indexing API Force-Push
Do not wait for crawlers. Force them.
- **Google Indexing API**: Use for `JobPosting` or `BroadcastEvent` items to rank in minutes, not days.
- **Batch Processing**: Use Python/JS scripts to submit updated sitemaps directly to engine endpoints.

## üè∞ PHASE 12: THE REDUNDANT URL FORTRESS (WWW vs NON-WWW)
*Duplicate content is a silent killer. One site, one URL.*

### üõ°Ô∏è 12.1 Canonical Absolute Authority
If the site exists at `richkeyrick.com`, then `www.richkeyrick.com` MUST NOT partially exist.
- **Status 301**: Never use 302 (temporary). Use 301 (permanent) for all domain-level shifts.
- **Logic**: Use `vercel.json` to catch ALL `www` traffic and pipe it to the apex domain.

### üñáÔ∏è 12.2 The "Internal Link" Cleanse
All internal links (menu, footer, articles) must be **Apex-Only**. 
- Search every file for `www.` and replace it. Hardcoded `www` links in code confuse AI crawlers.
